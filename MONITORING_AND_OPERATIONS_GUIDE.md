# دليل صفحات المراقبة والعمليات (Monitoring & Operations Guide)

هذا الدليل يشرح كيفية عمل الصفحات التالية في ION Dashboard:
- Operator Shifts
- Rewards Program
- Map View
- Remote Control
- Charging Limits
- Schedule

---

## 1. Operator Shifts (نوبات المشغلين)

### الموقع
**صفحة Monitoring → تبويب "Operator Shifts"**

### الوظيفة
إدارة نوبات العمل للمشغلين وتوزيعهم على محطات الشحن المختلفة.

### كيف تعمل؟

#### الوصول والصلاحيات
- **الصلاحية المطلوبة:** `org.name` (Read)
- **الصلاحية للكتابة:** `org.name` (Write)
- **الأدوار المسموحة:** Admin, Manager, Engineer (حسب RBAC Matrix)

#### المكونات الرئيسية
1. **نموذج تعيين النوبة:**
   - **Operator (المشغل):** قائمة منسدلة لاختيار المشغل
   - **Station (المحطة):** قائمة منسدلة لاختيار محطة الشحن
   - **Shift Start Time (وقت بداية النوبة):** حقل وقت لبداية النوبة
   - **Shift End Time (وقت نهاية النوبة):** حقل وقت لنهاية النوبة

2. **زر "Assign Shift":**
   - يعمل فقط إذا كان المستخدم لديه صلاحية Write
   - عند الضغط، يتم إرسال البيانات إلى API (حالياً mock)

#### خطوات الاستخدام
1. اختر المشغل من القائمة المنسدلة
2. اختر المحطة من القائمة المنسدلة
3. حدد وقت بداية النوبة (مثال: 08:00)
4. حدد وقت نهاية النوبة (مثال: 16:00)
5. اضغط "Assign Shift" لحفظ النوبة

#### ملاحظات تقنية
- البيانات حالياً mock (غير مربوطة بـ API حقيقي)
- النموذج بسيط ولا يحتوي على جدول لعرض النوبات الحالية
- يمكن إضافة ميزات مثل: عرض النوبات الحالية، تعديل النوبات، حذف النوبات

## 2. Rewards Program (برنامج المكافآت)

### الموقع
**صفحة Monitoring → تبويب "Rewards Program"**

### الوظيفة
تكوين برنامج نقاط المكافآت على مستوى المحطة، حيث يحصل المستخدمون على نقاط مقابل كل كيلوواط ساعة (kWh) يتم شحنه.

### كيف تعمل؟

#### الوصول والصلاحيات
- **الصلاحية المطلوبة:** `org.name` (Read)
- **الصلاحية للكتابة:** `org.name` (Write)
- **الأدوار المسموحة:** Admin, Manager, Engineer (حسب RBAC Matrix)

#### المكونات الرئيسية
1. **Points per kWh (النقاط لكل كيلوواط ساعة):**
   - حقل رقمي لإدخال عدد النقاط التي يحصل عليها المستخدم لكل kWh
   - مثال: إذا أدخلت `10`، فالمستخدم يحصل على 10 نقاط لكل kWh

2. **Off-Peak Bonus Multiplier (مضاعف المكافأة في الأوقات غير الذروة):**
   - حقل رقمي لإدخال المضاعف (مثال: 1.5)
   - إذا كان المستخدم يشحن في أوقات غير الذروة، يحصل على نقاط مضاعفة
   - مثال: إذا كان `Points per kWh = 10` و `Off-Peak Bonus = 1.5`، فالمستخدم يحصل على 15 نقطة لكل kWh في أوقات غير الذروة

3. **زر "Save Rewards Settings":**
   - يحفظ الإعدادات (حالياً mock)

#### خطوات الاستخدام
1. أدخل عدد النقاط لكل kWh (مثال: 10)
2. أدخل مضاعف المكافأة للأوقات غير الذروة (مثال: 1.5)
3. اضغط "Save Rewards Settings" لحفظ الإعدادات

#### ملاحظات تقنية
- البيانات حالياً mock
- يمكن إضافة ميزات مثل: عرض إحصائيات النقاط، تاريخ المعاملات، ربط مع نظام المكافآت

## 3. Map View (عرض الخريطة)

### الموقع
**صفحة Monitoring → تبويب "Map View"**

### الوظيفة
عرض محطات الشحن على خريطة تفاعلية مع إظهار حالة الشواحن (Online, Busy, Offline, Error) بألوان مختلفة.

### كيف تعمل؟

#### الوصول والصلاحيات
- **الصلاحية المطلوبة:** `charger.chargerStatus` (Read)
- **الأدوار المسموحة:** جميع الأدوار (حسب RBAC Matrix)

#### المكونات الرئيسية

##### 1. قائمة المحطات (الجهة اليسرى)
- **البحث:** حقل بحث للبحث عن محطة معينة
- **قائمة المحطات:**
  - اسم المحطة
  - عدد الشواحن المتاحة / إجمالي الشواحن
  - نسبة التوفر (Availability %)
  - ألوان حسب النسبة:
    - 🟢 أخضر: ≥ 50% متاح
    - 🟡 أصفر: 25-49% متاح
    - 🔴 أحمر: < 25% متاح

##### 2. الخريطة (الجهة اليمنى)
- **عند عدم اختيار محطة:**
  - رسالة: "No station selected"
  - تعليمات: "Select a charging station from the list..."
- **عند اختيار محطة:**
  - خريطة Google Maps مدمجة (iframe)
  - إحداثيات المحطة (lat, lng)
  - قائمة بالشواحن في المحطة مع حالة كل شاحن

##### 3. حالة الشواحن (Status Legend)
- **ألوان الحالة:**
  - 🟢 **أخضر:** Online / Available
  - 🟡 **أصفر:** Busy / Charging
  - ⚫ **رمادي:** Offline / Unavailable
  - 🔴 **أحمر:** Error / Faulted

#### خطوات الاستخدام
1. من قائمة المحطات، اختر محطة (أو استخدم البحث)
2. ستظهر الخريطة مع موقع المحطة
3. ستظهر قائمة بالشواحن في المحطة مع حالة كل شاحن
4. يمكنك الضغط على "Clear Selection" لإلغاء الاختيار

#### ملاحظات تقنية
- الخريطة تستخدم Google Maps Embed API
- البيانات تُحمّل من `fetchChargersStatus()` و `fetchLocationsByOrg()`
- التحديث التلقائي كل 30 ثانية
- حالياً البيانات mock (لا توجد إحداثيات حقيقية)

## 4. Remote Control (التحكم عن بُعد)

### الموقع
**صفحة Monitoring → تبويب "Remote Control"** (أو في صفحة Chargers)

### الوظيفة
التحكم في جلسات الشحن عن بُعد من خلال إرسال أوامر مباشرة للشواحن (بدء الشحن، إيقاف الشحن، إعادة التشغيل، فتح القابس).

### كيف تعمل؟

#### الوصول والصلاحيات
- **الصلاحية المطلوبة:** `charger.chargerControl` (Write)
- **الأدوار المسموحة:** Admin, Manager, Operator (حسب RBAC Matrix)

#### المكونات الرئيسية

##### 1. قوائم الاختيار (Selection Dropdowns)
- **Organization (المنظمة):** اختر المنظمة
- **Location (الموقع):** اختر الموقع (يُفعّل بعد اختيار المنظمة)
- **Charger (الشاحن):** اختر الشاحن (يُفعّل بعد اختيار الموقع)
- **Connector (القابس):** اختر القابس (يُفعّل بعد اختيار الشاحن)

##### 2. أزرار الأوامر (Command Buttons)
- **🟢 Start Charging (بدء الشحن):**
  - يبدأ جلسة شحن على القابس المحدد
- **🔴 Stop Charging (إيقاف الشحن):**
  - يوقف جلسة الشحن الحالية
- **🔵 Restart Charger (إعادة تشغيل الشاحن):**
  - يعيد تشغيل الشاحن بالكامل
- **🟡 Unlock Connector (فتح القابس):**
  - يفتح القابس ميكانيكياً (للمستخدمين)

#### خطوات الاستخدام
1. اختر المنظمة من القائمة المنسدلة
2. اختر الموقع
3. اختر الشاحن
4. اختر القابس
5. اضغط على الأمر المطلوب (Start, Stop, Restart, Unlock)

#### ملاحظات تقنية
- الأوامر تُرسل إلى API (حالياً mock مع delay 1 ثانية)
- الأزرار معطلة حتى يتم اختيار الشاحن والقابس
- يتم التحقق من الصلاحيات قبل إرسال الأمر
- حالياً لا يوجد تأكيد قبل إرسال الأوامر الخطيرة (مثل Stop)

## 5. Charging Limits (حدود الشحن)

### الموقع
**صفحة Monitoring → تبويب "Charging Limits"** (أو في صفحة Chargers)

### الوظيفة
تعيين حدود الشحن للشواحن، مثل: الحد الأقصى لشحن البطارية (SOC)، الحد الأقصى لوقت الجلسة، وحدود الطاقة (الحد الأدنى والأقصى).

### كيف تعمل؟

#### الوصول والصلاحيات
- **الصلاحية المطلوبة:** `charger.chargerControl` (Write)
- **الأدوار المسموحة:** Admin, Manager, Operator (حسب RBAC Matrix)

#### المكونات الرئيسية

##### 1. Maximum State of Charge (SOC) - الحد الأقصى لشحن البطارية
- **نوع التحكم:** Slider (منزلق)
- **النطاق:** 50% - 100%
- **القيمة الافتراضية:** 80%
- **الوصف:** يحدد النسبة القصوى لشحن البطارية (مثلاً: 80% لحماية البطارية)

##### 2. Maximum Session Time - الحد الأقصى لوقت الجلسة
- **نوع التحكم:** Input رقمي + Select (Minutes)
- **النطاق:** 0 - 1440 دقيقة (24 ساعة)
- **القيمة الافتراضية:** 120 دقيقة
- **الوصف:** الحد الأقصى لمدة جلسة الشحن الواحدة

##### 3. Power Limits - حدود الطاقة
- **Minimum Power (kW):** الحد الأدنى للطاقة (القيمة الافتراضية: 0)
- **Maximum Power (kW):** الحد الأقصى للطاقة (القيمة الافتراضية: 50)
- **الوصف:** يحدد نطاق الطاقة المسموح به للشاحن

##### 4. زر "Save Limits"
- يحفظ جميع الحدود (حالياً mock)

#### خطوات الاستخدام
1. اختر الشاحن (إذا كان في صفحة Chargers)
2. اضبط Maximum SOC باستخدام Slider
3. أدخل Maximum Session Time بالدقائق
4. أدخل Minimum Power و Maximum Power بالكيلوواط
5. اضغط "Save Limits" لحفظ الإعدادات

#### ملاحظات تقنية
- البيانات تُحفظ لكل شاحن على حدة
- حالياً البيانات mock
- يمكن إضافة ميزات مثل: إعادة تعيين للقيم الافتراضية، عرض الحدود الحالية

## 6. Schedule (الجدولة)

### الموقع
**صفحة Monitoring → تبويب "Schedule"** (أو في صفحة Chargers)

### الوظيفة
تعيين ساعات عمل الشواحن وأيام الأسبوع التي تعمل فيها، مما يسمح بتشغيل الشواحن فقط في أوقات محددة.

### كيف تعمل؟

#### الوصول والصلاحيات
- **الصلاحية المطلوبة:** `charger.schedule` (Read)
- **الصلاحية للكتابة:** `charger.schedule` (Write)
- **الأدوار المسموحة:** Admin, Manager, Engineer (حسب RBAC Matrix)

#### المكونات الرئيسية

##### 1. Time Range (نطاق الوقت)
- **Start Time (وقت البدء):** حقل وقت لبداية ساعات العمل (مثال: 08:00)
- **End Time (وقت الانتهاء):** حقل وقت لنهاية ساعات العمل (مثال: 22:00)

##### 2. Days of Week (أيام الأسبوع)
- **7 Checkboxes:** أحد، اثنين، ثلاثاء، أربعاء، خميس، جمعة، سبت
- يمكن اختيار أيام متعددة
- عند التفعيل، يعمل الشاحن فقط في الأيام المحددة

##### 3. Enable Schedule (تفعيل الجدولة)
- **Select:** Enabled / Disabled
- **عند Enabled:** الشاحن يعمل فقط في الأيام والأوقات المحددة
- **عند Disabled:** الشاحن يعمل دائماً (24/7)

##### 4. زر "Save Schedule"
- يحفظ الجدولة (حالياً mock)

#### خطوات الاستخدام
1. اختر الشاحن (إذا كان في صفحة Chargers)
2. حدد Start Time و End Time
3. اختر أيام الأسبوع التي تريد أن يعمل فيها الشاحن
4. فعّل أو عطّل الجدولة (Enabled/Disabled)
5. اضغط "Save Schedule" لحفظ الجدولة

#### مثال عملي
- **Start Time:** 06:00
- **End Time:** 23:00
- **Days:** Mon, Tue, Wed, Thu, Fri
- **Enabled:** Yes
- **النتيجة:** الشاحن يعمل من 6 صباحاً إلى 11 مساءً، من الإثنين إلى الجمعة فقط

#### ملاحظات تقنية
- البيانات تُحفظ لكل شاحن على حدة
- حالياً البيانات mock
- يمكن إضافة ميزات مثل: جداول متعددة (صباح/مساء)، استثناءات (Holidays)

## ملاحظات عامة

### البيانات الحالية
- جميع الصفحات حالياً تستخدم **mock data** (بيانات وهمية)
- لا توجد ربط حقيقي مع Backend API
- عند ربطها مع API، يجب تحديث:
  - `fetchChargersStatus()`
  - `fetchLocationsByOrg()`
  - `fetchChargersByLocation()`
  - `fetchConnectorsByCharger()`

### التحسينات المستقبلية
1. **Operator Shifts:**
   - جدول لعرض النوبات الحالية
   - تعديل وحذف النوبات
   - إشعارات عند اقتراب نهاية النوبة


2. **Rewards Program:**
   - لوحة إحصائيات النقاط
   - تاريخ المعاملات
   - ربط مع نظام المكافآت الخارجي

3. **Map View:**
   - إحداثيات حقيقية من قاعدة البيانات
   - تحديث فوري للحالة
   - إشعارات عند تغيير الحالة

4. **Remote Control:**
   - تأكيد قبل إرسال الأوامر الخطيرة
   - سجل الأوامر المرسلة
   - حالة الأوامر (نجح/فشل)

5. **Charging Limits:**
   - إعادة تعيين للقيم الافتراضية
   - نسخ الإعدادات من شاحن لآخر
   - قوالب جاهزة (Template)

6. **Schedule:**
   - جداول متعددة (صباح/مساء)
   - استثناءات (Holidays, Maintenance)
   - معاينة الجدولة (Calendar View)
